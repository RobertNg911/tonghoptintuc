# Phase 3 - CONTEXT.md

## Phase Overview

**Phase 3: Facebook Integration** — Đăng bài lên Fanpage với token management

## Decisions Made

### 1. Token Management

**Token Type**: Page Access Token (NOT user token)

**How to get**:
1. Go to [Graph API Explorer](https://developers.facebook.com/tools/graph-api/)
2. Select your App → Get Token → Select Page
3. Copy the Page Access Token
4. Store in `FACEBOOK_ACCESS_TOKEN` env var

**Token Expiry**: 60 days (Facebook policy)

**Renewal Strategy**:
- Manual renewal initially (update env var)
- Auto-renewal in v2 if needed (requires long-lived token flow)

### 2. Post Format

**Format**: Photo post (text + image)
- Better engagement than link-only posts
- Image generated by AI (Phase 5) or skip if unavailable

**Structure**:
```
[Hook/Title]

[Content - 150-200 words]

[Hashtags]

[Source: via @SourceName]
```

**Caption Limit**: 63,206 characters (Facebook max), but aim for 500-1000 for readability

### 3. API Endpoints

**Post to Page**:
```
POST /{page-id}/photos
or
POST /{page-id}/feed
```

**Get Page Info**:
```
GET /{page-id}?fields=id,name,fan_count
```

### 4. Spam Prevention

**Timing**: Random ±10-15 minutes from scheduled time
- Cron triggers at :00
- Actual post: :00 + random(0, 15) minutes

**Content Variation**:
- Different hook each post
- Vary format slightly (occasionally no image, text only)

**Rate Limiting**:
- Max 1 post/hour (matches scheduling)
- No retry on rate limit error

### 5. Error Handling

| Error | Action |
|-------|--------|
| Invalid token | Log error, alert admin, stop pipeline |
| Rate limit | Wait, retry once after 5 min |
| Missing photo | Fallback to text-only post |
| API timeout | Log, continue with next step |

### 6. Environment Variables

```env
FACEBOOK_PAGE_ID=your_page_id
FACEBOOK_ACCESS_TOKEN=your_page_access_token
```

### 7. Implementation

**Location**: `src/services/facebook.ts`

**Functions**:
- `postToPage(caption: string, imageUrl?: string): Promise<PostResult>`
- `getPageInfo(): Promise<PageInfo>`
- `validateToken(): Promise<boolean>`

## Code Context

- **Framework**: Hono (from Phase 1)
- **API**: Facebook Graph API v25.0
- **HTTP Client**: Built-in fetch (Cloudflare Workers)

## Prior Decisions Applied

- No database (from constraints)
- Randomize posting time (from PITFALLS.md)
- Page token, not user token (from research)

## Pitfalls to Avoid (from research)

1. **Wrong token type** → Use Page Access Token
2. **Token expiration** → Renew before 60 days
3. **Spam detection** → Randomize timing, vary content

## Out of Scope for Phase 3

- AI image generation (Phase 5)
- Scheduling (Phase 4)
- Multiple platforms (v2)
